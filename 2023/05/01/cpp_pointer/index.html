C++-Pointer

發表於2023-05-01|更新於2023-05-02|計算機語言
|閱讀量:

基本概念

變量地址

  • 用&變量名可以獲取地址
  • 烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫烫
    • 因為c++在獲取char類型地址有bug
  • 解決方法

指針變量

  • 是一種特殊變量，專用於存放變量在內存中的起始地址
  • 語法
    • 數據類型 *變量名
  • 起始地址意思為，如果有一個4字節的int變量，它佔用了這一塊內存，4個字節，起始字節就是0x00000008

對指針賦值

  • 無論什麼類型的數據地址都是十六進制數
  • 用整型指針存放整數型變量的地址，用字符型指針存放字符型變量的地址
  • 注意
    • 對指針的賦值操作也通俗稱為“指向某變量”，被指向的變量的數據類型稱為“基類型”
    • 如果指針的數據類型與基類型不符，編譯會出現警告，但是，可以強制轉換他們的類型。

指針佔用的內存

  • 64位系統中，佔用8字節
  • 32位系統中，佔用4字節

使用指針

  • 在聲明指針後，沒有賦初始值，不能使用
  • 指針存放變量的地址，因此，指針名表示的是地址
  • *為解引用運算符

  • 一個變量可以被多個指針引用
  • 程序在存儲數據的時候，必須跟蹤三種屬性
    • 數據存儲在哪裡
    • 數據是什麼類型
      • 用處：決定佔用內存大小，因為指針記錄了起始位置，有佔用內存大小，才能決定系統如何操作數據
    • 數據的值是多少
  • 兩種策略可以達到以上目的
    • 聲明一個普通變量，聲明時指出數據類型和變量名，系統在內部跟蹤該內存單元
    • 聲明一個指針變量，存儲的值是地址，不是值本身，程序直接訪問該內存單元

指針用於函數的參數

  • 把地址的形參聲明為指針，調用的時候把實參的指針傳入，形參存放的是實參的地址，在函數中通過解引用的方式直接操作內存的數據，可以修改實參的值，稱為地址傳遞或者傳地址
  • 值傳遞
    • 函數的實參是普通變量
  • 意義
    • 可以在函數中修改實參的值
    • 可以減少內存拷貝，提升性能

const 修飾指針

常量指針

  • 語法
    • const 數據類型 *變量名
  • 不能通過解引用的方法修改內存地址中的值
  • 注意
    • 指向的變量（對象）可以改變（之前是指向變量a的，後來可以改為指向變量b）
    • 一般用於修飾函數的形參，表示不希望在函數裡修改內存地址的值

指針常量

  • 語法
    • 數據類型 * const 變量名
  • 注意
    • 在定義的同時必須初始化，否則沒有意義
    • 可以通過解引用的方法修改內存地址的值
    • 指向的對象不可改變
    • C++把指針常量改為引用

常指針常量

  • 語法
    • const 數據類型 * const 變量名
  • 常引用

總結

  • 常量指針
    • 指針指向可以改，值不能改
  • 指針常量
    • 指針指向不能改，值可以改
  • 常指針常量
    • 指針指向不能改，值不能改

void關鍵字

  • void*能接受任意類型的地址
  • 不能對void*指針直接解引用，需要轉換為其他指針類型
  • 把其他指針賦值給void*不需要轉換
  • 把void*指針賦值給其他指針需要轉換

動態分配內存

  • 使用堆區內存四個步驟
  1. 聲明一個指針
  2. 用new運算符向系統申請一塊內存，讓指針指向這塊內存
  3. 通過對指針解引用的方法，想使用變量一樣使用這塊內存
  4. 如果這塊內存不用了，用delete運算符釋放它
  • 申請內存的方法
    • new 數據類型(初始值);
    • 申請成功則返回一個地址
    • 失敗則返回一個空地址
  • 釋放內存方法
    • delete 地址

注意

- 動態分配的變量沒有變量名，只能通過指向它的指針操作內存中的數據
- 如果動態分配的內存不用了，就用delete釋放他，否則可能用盡系統內存
- 動態分配的內存聲明週期與程序相同，程序退出時，如果沒有釋放，系統將自動回收
- 就算指針的作用域已失效，所指向的內存也不會釋放
- 用指針跟蹤已分配的內存，不能跟丟

二級指針

  • 指針用於存放普通變量的地址
  • 二級指針用於存放指針變量的指針
  • 聲明二級指針的語法
    • 數據類型** 指針名
  • 目的
    1. 傳遞地址
    2. 存放動態分配的內存的地址

空指針

野指針

  • 指針指向的不是一個有效的地址
  • 訪問野指針可能會造成程序崩潰
  • 出現野指針的情況主要有三種
    1. 在定義的時候，如果沒有初始化，它的值是不確定的
    2. 如果用指針指向了動態分配的內存，內存被釋放後，指針不會置空，但是指向的地址已經失效
    3. 指針指向的變量已超越變量作用域（變量的內存空間已被系統回收）
  • 規避方法
    1. 指針在定義的時候，沒地方指，就指nullptr
    2. 動態分配的內存釋放後，將其置為nullptr
    3. 函數不要返回局部變量的地址
  • 注意
    • 野指針危害大，因為不一定造成崩潰，增加了調試程序的難度

一維數組和指針

指針的算數

  • 一個integer加1，其指將加1
  • 指針變量加1，增加的量等於它指向的數據類型的字節數

數組的地址

  1. 數組在內存中佔用的空間是連續
  2. C++將數組名截石位數組第0個元素
  3. 數組第0個元素的地址與首地址的取值相同
  4. 數組第n個元素的地址是
    • 數組首地址+n
  5. C++編譯器把數組名[下標]解釋為*(數組首地址+下標)

數組本質

  • 數組是佔用連續空間的一塊內存，數組名被解釋為數組第0個元素地址
  • C++操作這塊內存有兩種方法
    • 數組解釋法
    • 指針表示法

數組名不一定被解釋為地址

  • 多數情況下，C++將數組名解釋為數組第0個元素地址，但是，sizeof運算符用於數據名時，將返回整個數組佔用內存空間的字節數
  • 可以修改指針的值，但是數組名是常量，不可修改
