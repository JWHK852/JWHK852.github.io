Django2-模型

發表於2022-12-23|更新於2023-06-23|計算機語言
|閱讀量:

ORM

  • 對象-關係-映射
  • 任務
    • 根據對象的類型生成表結構
    • 將對象，列表的操作轉換為sql語句
    • 將sql語句查詢的結果轉換為對象，列表
  • 優點
    • 極大地減輕開發人員工作量，不需要因為數據庫的變更修改代碼

定義模型

  • 模型，屬性，表，字段之間的關係
    • 一個模型類在數據庫中對應一張表，在模型類中定義的屬性，對應該模型對照表中的一個字段

定義屬性

  • Django根據屬性的類型確定一下信息
    • 當前選擇的數據庫支持的字段類型
    • 渲染管理表單使用的默認HTML控件
    • 在管理站點最低限度的驗證

元選項

  • 在模型中定義一個meta類用於定義源信息

db_table

  • 定義數據表名，推薦使用小寫字母，數據表名默認為項目名_類目

ordering

  • 對象的默認排列字段，獲取對象的列表時使用
  • ordering["id"]升序
  • ordering["-id"]降序
  • e.g.

模型成員

類屬性

- objects
    - manager類型的一個對象，作用是與數據庫進行交互
    - 當定義模型類是沒有指定管理器，則Django為模型創建一個名為objects的管理器
- 自定義模型管理器
- 當自定義模型管理器，objects就不存在

- 自定義管理器Manager類
    - 模型管理器是Django的模型進行與數據庫交互得接口，一個模型可有多個模型管理器
    - 作用
        - 向管理器類中添加額外的方法
        - 修改管理器返回的原始查詢集
            - 重寫get_queryset()方法
    - 例子
        - 把isDeleted為Ture的數據過濾

創建對象

  • 目的

    • 向數據庫添加數據

  • 當創建對象時，Django中不會對數據進行讀寫操作，當調用save()方法時才與數據庫交互，將對象保存到數據表中

  • 注意

    • __init__方法在父類models.Model中使用，在自定義模型中無法使用

  • 方法

    1. 在模型類中添加一個類方法
    • 傳參創建對象
    1. 在定義管理器中添加一個方法

模型查詢

概念

  • 查詢集表示從數據庫獲取的對象集合
  • 查詢集可以有多個過濾器
  • 過濾器就是一個函數，基於所給的參數限制查詢集結果
  • 從SQL角度，查詢集和select語句一樣，過濾器就如where條件

查詢集

  • 在管理器上調用過濾器方法返回查詢集

  • 查詢集經過過濾器篩選後返回新的查詢集，所以可以寫為鏈式調用

  • 惰性執行

    • 創建查詢集不會帶來任何數據的訪問，直到調用數據才會訪問數據

  • 直接訪問數據的情況

    • 迭代
    • 序列化
    • 與if合用

  • 返回查詢集的方法為過濾器

    • all()
      • 返回查詢集中的所有數據
    • filter()
      • 返回符合條件的數據
      • filter(鍵=值)
      • filter(鍵=值, 鍵=值)
        • filter(鍵=值).filter(鍵=值)
    • exclude()
      • 過濾符合條件的數據
    • order_by()
      • 排序
    • values()
      • 一條數據就為一個對象(字典)，返回一個列表

  • 返回單個數據

    • get()
      • 返回一個滿足條件的對象
      • 注意
        • 如果沒有找到符合條件的對象，會引發模型類.DoesNotExist異常
        • 如果找到多個對象，會引發模型類.MultipleObjectReturned異常
    • count()
      • 返回當前查詢集中返回對象的個數
    • first()
      • 返回當前查詢集中的第一個返回對象
    • last()
      • 返回當前查詢集中的最後一個返回對象
    • exists()
      • 判斷查詢集中是否有數據，有返回True，沒有返回False

  • 限制查詢集

    • 查詢集返回列表，可以使用下標的方法進行限制，等於SQL中的limit語句
    • 注意：下標不能為負數
    • studentsList = grade.students_set.all()[0:5]

  • 查詢集的緩存

    • 概述
      • 每一個查詢集都包含一個緩存，來最小化的對數據庫訪問
      • 在新建的查詢集中，緩存首次為空，第一次對查詢值求值，會發生數據緩存，Django會將查詢出來的數據做一個緩存，並返回查詢結構，以後的查詢直接使用查詢緩存

  • 字段查詢

    • 概述
      • 實現了SQL中的where語句，作為方法filter(),exclude(),get()的參數
      • 語法
        • 屬性名稱__比較運算符=值
      • 外鍵
        • 屬性名_id
      • 轉義
        • SQL中like語句中使用%匹配佔位，匹配數據中的%需要轉義where like \%
        • filter(sname_contains='%')
        • Django中直接可以匹配%
    • 比較運算符
      • exact
        • 判斷，大小寫敏感
        • filter(isDeleted=false)
      • contains
        • 是否包含，大小寫敏感
        • studentsList = Students.stuObj2.filter(sname__contains="劉")
          • 選擇所有姓劉的學生
      • startswith, endswith
        • 以value開頭或者結尾，大小寫敏感
      • 以上4個與前面加上i就不區分大小寫
        • iexact,icontains,istartswith.iendswith
      • isnull, isnotnull
        • 是否為空
        • filter(sname__isnull=true)
      • in
        • 是否包含在範圍內
        • studentsList = Students.stuObj2.filter(pk__in=[1,2,3,4,5])
      • gt,gte,lt,lte
        • 大於，大於等於，小於，小於等於
        • studentsList = Students.stuObj2.filter(sage__gte=30)
      • year,month,day,week_day,hour,minute,second
        • studentsList = Students.stuObj2.filter(lastTime__year=2022)
      • 跨關聯查詢
        • 處理join查詢
          • 語法
            • 模型類名_屬性名_比較運算符
          • 例子
            • scontend中帶有Jerry的對象的班級屬性
            • grade = Grades.objects.filter(students__scontend__contains='Jerry')
      • 查詢快捷
        • pk
          • 代表主鍵
    • 聚合函數
      • 使用aggregate()函數返回聚合函數的值
      • Avg
      • Count
      • Max
      • Min
      • Sum
    • F對象
      • 可以使用模型的A屬性與B屬性進行比較
        • 例子：取出女生比男生數量多的班級
      • 支持F對象的算數運算
        • 例子：取出女生比（男生數量+20）多的班級
    • Q對象
      • 概述
        • 過濾器的方法中的關鍵字參數，條件為AND模式
      • 需求
        • 進行OR查詢
      • 解決
        • 使用Q對象
      • 例子
        • 取出主鍵小於或等於3 或 年齡大於50的學生對象
      • 只有一個Q的情況
        • 等於普通匹配
        • 加上~為取反的意思
