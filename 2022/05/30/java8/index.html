封裝

  • 一般對成員變量使用private（私有、隱藏）關鍵字修飾（private修飾的成員只能在當前類中訪問），對外不能直接訪問

  • 為每個成員變量提供配套public修飾的getter、setter方法暴露其取值和賦值

  • Student.java

HOTKEY

  • 在輸入完private變量後，按Alt + Insert打开Generator，選擇Getter and Setter，選取所有變量後點擊OK

static 關鍵字

  • static 為靜態的意思，可以修飾成員變量和成員方法
  • static 修飾成員變量表示該成員變量只在內存中儲存一份，可以被共享訪問和修改
  • 靜態成員變量（有static修飾，屬於類，內存中加載一次）
    • 常表示如在線人數信息等需要被共享的信息，可以被共享訪問
  • 實例成員變量
    • 屬於每個對象，且每個對象的信息不同（如：age，name…）

static 修飾成員方法

  • 靜態成員方法，建議用類名訪問
  • 實例成員方法，只能用對象訪問

注意事項

  • 靜態方法只能訪問靜態成員，不可以直接訪問實例成員
  • 實例方法可以訪問靜態成員，也可以直接訪問實例成員
  • 靜態方法不可以出現this關鍵字

繼承

  • java中提供一個關鍵字extends，可以讓一個類和別的類
  • Student稱為子類，People稱為父類
  • 好處
    • 提高代碼的複用性，減少代碼冗餘，增強類的功能拓展性
  • 特點
    • 子類繼承父類，子類可以得到父類的屬性和行為，子類可以使用

設計規範

  • 子類相同方法應該放在父類中定義，子類獨有的屬性和行為應該定義在子類自己裡面

繼承特點

  • 子類可以繼承父類的屬性和行為，但是不能繼承父類的構造器
    • 子類有自己的構造器，父類構造器用於初始化父類對象
  • java是單繼承模式，一個子類只可以繼承一個直接父類
  • java不支持多繼承，但是支持多層繼承
  • java中所有的類都是object類的子類
  • 子類可以直接使用父類的靜態成員
  • 子類不可以直接訪問父類私有成員
  • 子類方法中訪問成員（成員方法，成員變量）滿足就近原則
    • 現在子類局部範圍查找
    • 然後再子類成員範圍查找
    • 然後再父類成員範圍查找
  • 如果子類父類出現重名的成員，會優先使用子類的，但是如果一定要使用父類的，可以使用super關鍵字訪問指定父類成員

方法重寫

  • 在繼承體系中，子類出現了和父類一樣的方法聲明，這個子類的方法是重寫的方法
  • 應用場景
    • 如果子類需要父類的功能，但父類的該功能不能完全滿足需求
    • 子類可以重寫父類中的方法

@Override 重寫註解

  • @Override放在重寫後的方法上，作為重寫是否正確的校驗註解
  • 提高程序的可讀性

注意事項

  • 重寫方法的名稱，形參列表必須與被重寫方法的名字和形參列表一樣
  • 私有方法不可重寫
  • 子類重寫父類方法，訪問權限必須大於或等於父類
    • private < 缺省 < protected < public
  • 靜態方法無法重寫
