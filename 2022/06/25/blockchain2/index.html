前置

Ganache-cli

  • 用於在本地模擬ETH BLOCKCHAIN
  • Download地址
  • 點擊Address右側的鑰匙圖標可以顯示地址與私鑰
  • Ganache 命令行指令
  1. 安裝nodejs
  2. 安裝yarn
    • npm install --global yarn
  3. yarn global add ganache-cli
  4. CMD輸入ganachi-cli

Python

  • 如果希望部署于Rinkeby，更改中間的w3，chainid，address，private_key
  • chainid獲取
  • 與metamask內查看address與private key
  • 註冊一個API地址
  • 複製該網址，稍後輸入進Web3.HTTPProvider
  • 修改前
  • 修改後

Brownie

  • 方便用戶部署contract

安裝

  1. python -m pip install --user pipx
  2. python -m pipx ensurepath
  3. pipx install eth-brownie

初始化brownie

  • 建立一個文件夾，並且在文件夾內的cmd內輸入

  • Compile Contract

給brownie添加私鑰

  • brownie accounts new 賬號名稱

  • 輸入私鑰

  • 輸入密碼

  • 查看已有賬戶列表

    • brownie accounts list

Script

  • 在script文件夾內新建deploy.py

使用Brownie部署合約

  • Brownie 中無須聲明方法的屬性

本地模拟器部署

測試網絡部署

  • 使用命令brownie networks list可以顯示brownie支持的網絡
  • 於.env文件內加入export WEB3_INFURA_PROJECT_ID = xxxx，xxxx填入Infura Project ID

使用.env需要在根目錄加入brownie-config.yaml文件

  • 如果需要只測試一個函數，則運行brownie test -k 函數名
    • 例如brownie test -k test_updating_storage
  • 進入調試模式brownie test --pdb，用於調試錯誤（只會在出錯時才可以進入pdb模式）
  • 使用brownie test -s可以顯示每個函數是否通過

讀取

  • 創建read_value.py
  • 輸入指令brownie run scripts/read_value.py --network rinkeby

Brownie Console

  • 一個包含智能合約的Python console

Brownie Fund Me

  • 使用之前的Fund Me會發現Brownie會對import報錯，因為solidity無法識別npm導入
  • 需要尋找github版本並導入
  1. 創建brownie-config.yaml文件
  2. 按照- <organization/repo>@<version>格式輸入github包名

注意，brownie-config.yaml內不要有空的欄位，否則回報TypeError: argument of type 'NoneType' is not iterable錯誤

Brownie 交互

  • Brownie可以將python的參數傳入sol中
  • 上述例子中，在deploy_fund_me()函數的fund_me變量傳入了一個AggregatorV3Interface的參數，也就是usd轉eth的鏈接，而此變量會傳入FundMe.sol的constructor中
