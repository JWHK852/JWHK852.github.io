爬蟲簡介

什麼是爬蟲

  • 網絡爬蟲，可以自動化瀏覽網絡中的信息，當然瀏覽信息的時候需要按照我們制定的規則進行，這些規則成為網絡爬蟲算法。Python可以很方便地寫出爬蟲程序，進行互聯網信息的自動化檢索

為什麼學習爬蟲

  • 私人訂製一個搜索引擎，並且可以對搜索引擎的數據採集工作原理進行更深層次地理解
  • 獲取更多的數據源，並且這些數據源可以按我們的目的進行採集，去掉很多無關的數據
  • 更好地進行SEO（搜索引擎優化）

網絡爬蟲的組成

控制節點

  • 叫作爬蟲的中央控制器，只要負責根據url地址分配線程，並調用爬蟲節點進行具體的爬行

爬蟲節點

  • 按照相關的算法，對網頁進行具體的爬行，主要包括下載網頁以及對網頁的文本處理，爬行後會將對應的爬行結果儲存到對應的資源庫中

資源庫構成

  • 存儲爬蟲爬去到的響應數據，一般為數據庫

爬蟲設計思路

  • 首先確定爬取的網頁URL地址
  • 通過HTTP協議開獲取對應的HTML頁面
  • 提取HTML頁面裡有用的數據
    • 如果是需要的數據就保存起來
    • 如果頁面裡是其他的URL，那麼就繼續執行第二步

為什麼選擇Python做爬蟲

  • PHP
    • 雖然是世界上最好的語言，但是天生不是干爬蟲的命，PHP對多線程，異步支持不足，並發不足，爬蟲是工具性程序，對速度和效率要求較高
  • Java
    • 生態圈完善，是Python最大的對手。但是Java本身很笨重，代碼量大。重構成本比較高，任何修改都會導致大量的代碼變動。最攞命是爬蟲需要經常修改部分代碼。
  • C/C++
    • 運行效率和性能幾乎最強，但是學習成本非常高，代碼成型較慢，能用C/C++寫爬蟲，說明能力很強，但是不是最正確的選擇
  • Python（拍馬屁）
    • 語法優美，代碼簡潔，開發效率高，三方模塊多，調用其它接口也方便。有強大的爬蟲Scrapy，以及成熟高效的scrapy-redis分佈式策略

Python基本語法

  • 如題

如何抓取頁面

  • HTTP請求處理，rellib處理後的請求可以模擬瀏覽器發送請求，獲取服務器響應的文件

解析服務器響應的內容

  • re,xpath,BeautifulSoup4,jsonpath,pyquery
  • 目的是使用某種描述性語法來提取匹配規則的數據

如何採集動態HTML，驗證碼的處理

  • 通用的動態頁面採集
    • Selenium+PhantomJS（無界面瀏覽器），模擬真實瀏覽器加載js，ajax等非靜態頁面數據
  • Tesseract
    • 機器學習庫，機器圖像識別系統（識別圖片中的文本）

Scrapy框架

  • 常見的有Scrapy，Pyspider
  • 高定制性高性能（異步網絡框架twisted），所以數據下載速度非常快，提供了數據儲存，數據下載，提取規則等組件

爬蟲與反爬蟲與反反爬蟲

反爬蟲技術

  • User-Agent
  • 代理
  • 驗證碼
  • 動態數據加載
  • 加密數據

網絡爬蟲類型

通用網絡爬蟲

概念

  • 搜索引擎用的爬蟲系統

用戶群體

  • 搜索引擎

目標

  • 盡可能把互聯網上所有的網站下載下來，放到本地服務器裡形成備份。再對這些網頁做相關處理（提取關鍵字，去掉廣告等），最後提供一個用戶檢測接口

抓取流程

  • 首先選取一部分已有的URL，把這些URL放到待爬隊列
  • 從隊列裡提取這些URL，然後解析DNS的到主機IP，然後去這個IP對應的服務器下載HTML界面，保存到搜索引擎的本地服務器裡，之後把爬過的URL放入已爬去隊列
  • 分析這些網頁內容，找出網頁裡的URL，繼續執行第二步，直到爬取條件結束

搜索引擎如何獲得一個新網站的URL

  • 主動向搜索引擎提交網址
  • 在其他的網站裡添加網站的外鏈接
  • 搜索引擎會和DNS服務商合作，可以快速收錄新的網站

爬蟲不是萬物皆可爬，需要遵守規則

  • Robots協議
    • 協議會指明通用爬蟲可以爬去網頁的權限
    • Robots.txt只是一個建議，並不是所有的爬蟲都遵循，一般只有大型的搜索引擎爬蟲才會遵循

通用爬蟲工作流程

  • 爬去頁面
  • 存儲數據
  • 內容處理
  • 提供檢索/排名服務

搜索引擎排名

  • PageRank值
    • 根據網頁的流量（點擊量，瀏覽值，人氣）統計，流量越高，網站越值錢，排名越靠前
  • 競價排名
    • （花錢買榜。。。）

通用爬蟲的缺點

  • 只能提供和文本相關的內容（HTML，Word，PDF）等，但是不能提供多媒體（音樂，影片，圖片）和二進制文件（程序，腳本）等
  • 提供的結果千篇一律，不能針對不同人群提供不同的搜索結果
  • 不能理解人類語義上的檢索

聚焦網絡爬蟲

概念

  • 爬蟲程序員寫的針對某種內容的爬蟲

特點

  • 面向主題爬蟲，面向需求爬蟲
    • 會針對某種特定的內容去爬取信息，而且會保證信息和需求盡可能相關

其他爬蟲

  • 增量式網絡爬蟲
  • 深層網絡爬蟲

URL

  • 統一資源定位符，是互聯網上的資源地址
