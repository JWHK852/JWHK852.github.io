數據庫簡介

概念

  • 計算機誕生後，數據開始在計算機內儲存並計算，並設計出了數據庫系統
  • 數據庫解決的問題
    • 持久化儲存
    • 優化讀寫
    • 保證數據的有效性
  • 當前使用的數據庫主要分為兩類
    • 文檔型
      • 例如sqlite，通過對文件的複製完成數據庫的複製
      • 例如mysql，postgre，數據儲存在一個物理文件中，但是需要終端以tcp/ip協議連接，進行數據庫的讀寫操作

三範式

  • 經過研究和對使用中的問題的總結，對於設計數據庫提出了一些規範，這些規範稱為範式

範式

  • 第一範式（1NF）：列不可拆分
  • 第二範式（2NF）：唯一標識
  • 第三範式（3NF）：引用主鍵（唯一，不能相同，不能為空）

E-R模型

  • 當前物理數據庫都是按照E-R模型進行設計的
  • E代表entry，實體
  • R代表relationship，關係
  • 一個實體轉換為數據庫中的一個表
  • 關係描述兩個實體間的對應規則，包括
    • 一對一
    • 一對多
    • 多對多
  • 關係轉換為數據庫表中的一個列 *在關係型數據庫中，一行就是一個對象

主要操作

  • 數據庫的操作，包括創建，刪除
  • 表的操作，包括創建，修改和刪除
  • 數據的操作，包括增加，檢索，更新，刪除，簡稱crud

數據完整性

概念

  • 一個數據庫就是一個完整的業務單元，可以包括多張表，數據被儲存在表中
  • 在表中為了更加準確的存儲數據，保證數據的正確性，可以在創建表的時候，為表添加一些強制性的驗證，包括數據字段的類型，約束

字段類型

  • 數字：int，deciaml
    • deciaml(5, 2)表示一共有五位數，小數包含兩位
  • 字符串：char，varchar，text
  • 日期：datetime
  • 布爾：bit

約束

  • 主鍵primary key
    • 主鍵只能有一個
  • 非空not null
  • 唯一unique
    • 唯一可以有多個，物理上儲存的結構是由主鍵維護的
  • 默認default
  • 外鍵foreign key

命令

基本命令

啟動服務

  • 說明：以管理員身份運行cmd
  • 格式：net start 服務名稱
  • 示例：net start mysql80

停止服務

  • 說明：以管理員身份運行cmd
  • 格式：net stop 服務名稱
  • 示例：net stop mysql80

連接數據庫

  • 格式：mysql -u 用戶名 -p
  • 示例：mysql -u root -p & 輸入密碼

斷開連接

  • quit或exit

查看版本（連接後執行）

  • 示例：select version();

顯示當前時間（連接後執行）

  • 示例：select now();

遠程連接

  • 格式：mysql -h ip地址 -u 用戶名 -p & 輸入對方的MySQL密碼

數據庫操作命令

創建數據庫

  • 格式：create database 數據庫名 charset=utf8;
  • 示例：create database jerry charset=utf8;

刪除數據庫

  • 格式：drop database 數據庫名;
  • 示例：drop database jerry;

切換數據庫

  • 格式：use 數據庫名
  • 示例：use jerry

查看當前使用的數據庫

  • 格式：select database();

表操作命令

查看數據庫中所有的表

  • 格式：show tables;

創建表

  • 格式：create table 表名(列以及類型);
  • 說明：
    1. auto_increment 表明自增長
    2. primary key 表明主鍵
    3. not null 表示不為空
    4. default 聲明默認值
  • 示例：create table student(id int auto_increment primary key, name varchar(20) not null, age int not null, gender bit default 1, address varchar(20), isDelete bit default 0);

刪除表

  • 格式：drop table 表名;
  • 示例：drop table student;

查看表結構

  • 格式：desc 表名;
  • 示例：desc student;

查看建表語句

  • 格式：show create table 表名;
  • 示例：show create table student;

重命名表

  • 格式：rename table 原表名 to 新表名
  • 示例：rename table student to newStudent

修改表

  • 格式：alter table 表名 add|change|drop 列名 類型;
  • 示例：alter table student add p1 char;

數據操作命令

增加數據

全列插入

  • 格式：insert into 表名 values(...);
  • 說明：主鍵列是自動增長，但是在全列插入時需要佔位，通常使用0，插入成功以後以實際數據為準
  • 示例：insert into student values(0,"Irene",17,0,"France","");

缺省插入

  • 格式：insert into 表名(列1,列2,...) values(值1,值2,...);
  • 示例：insert into student(name,age,address) values("Prinz",17,"Germany")
  • 說明：id與gender有默認值，所以不用插入

同時插入多條數據

  • 格式：insert into 表名 values(...),(...),...;
  • 示例：insert into student values(0,"Candy",17,0,"HK"),(0,"Mandy",17,0,"HK"),(0,"Kelly",17,0,"HK");
  • 說明：

刪除數據

  • 格式：delete from 表名 where 條件;
  • 示例：delete from student where id=3;
  • 注意：沒有條件是全部刪除，慎用

修改數據

  • 格式：update 表名 set 列1=值1,列2=值2,... where 條件;
  • 示例：update student set 列1=值1,列2=值2,... where id=8;
  • 注意：沒有條件是全部列都修改，慎用

查詢數據

基本語法

  • 格式：select * from 表名;
  • 說明：
    1. from關鍵字後面是表名，表示數據來源於這張表
    2. select後面寫表中的列名，如果是*表示在結果集中顯示表中的所有列
    3. 在select後面的列名部分可以使用as為列名起別名，別名顯示在結果集中
    4. 如果要查詢多個列，之間使用,分隔
  • 示例：
    • select name,age from student;
    • select * from student;
    • select name as n, age from student;

消除重複行

  • 在select後面列的前面使用distinct可以消除重複的行
  • 示例：select distinct gender from student;

查看條件

語法
  • select * from 表名 where 條件
比較運算符

  • 等於

    • =

  • 大於

    • >

  • 小於

    • <

  • 大於等於

    • >=

  • 小於等於

    • <=

  • 不等於

    • != 或 <>

  • 示例：查詢id值大於8的所有數據

    • select * from student where id \> 8;
邏輯運算符
  • 並且
    • and
  • 或者
    • or
  • 非
    • not
  • 示例：查詢id值大於7的女同學
    • select * from student where id>7 and gender=0;
模糊查詢

  • like

    • %：表示任意多個任意字符
    • _：表示一個任意字符

  • 示例：查詢姓李的同學

    • select * from student where name like "李%";
    • select * from student where name like "李_";

範圍查詢

  • in：表示在一個非連續的範圍內

  • between...and...：表示在一個連續的範圍內

  • 示例1：查詢編號為8，10，12的學生

    • select * from student where id in (8,10,12);

  • 示例2：查詢編號為6-8的學生

    • select * from student where id between 6 and 8;
空判斷
  • 注意：null與""是不同的
  • 判斷空：is null
  • 判斷非空：is not null
  • 示例1：查詢沒有地址的同學
  • select * from student where address is null;
  • 示例2：查詢有地址的同學
  • select * from student where address is not null;
優先級
  • (),not,比較運算符，邏輯運算符
  • and比or優先級高，如果同時出現並希望先選or，需要結合()使用

聚合

  • 為了快速的到統計的數據，提供了五個聚合函數
  1. count(*)
    • 表示計算總行數，括號中可以寫*和列名
  2. max(列)
    • 表示求此列的最大值
  3. min(列)
    • 表示求此列的最小值
  4. sum(列)
    • 表示求此列的和
  5. avg(列)
    • 表示求此列的平均值
  • 示例：

分組

  • 按照字段分組，表示此字段相同的數據會被放到一個集合中

  • 分組後只能查詢出相同的數據列，對於有差異的數據列，它無法顯示在結果集中

  • 可以對分組後的數據進行統計做聚合運算

  • 格式：select 列1,列2,聚合... from 表名 group by 列1,列2,列3,...;

  • 示例1：查詢男女生總數，並以性別分組

  • select gender,count(*)from student group by gender;

  • 示例2：查詢男女生總數，並以性別和年齡分組

  • select name,gender,count(*)from student group by gender,age;

  • 篩選分組後的數據

    • 格式：select 列1,列2,聚合... from 表名 group by 列1,列2,列3,... having 列1,...,聚合,...;
    • 示例
    • select gender,count(*) from student group by gender having gender;

  • where 與 having 的區別：

    • where是對from後面指定的表進行篩選，屬於對原始數據的篩選
    • having是對group by的結果進行篩選

排序

  • 格式：select * from 表示 order by 列1 asc|desc,列2 asc|desc,...;

  • 說明

    1. 將數據按照列1進行排序，如果某些列1的值相同，則按照列2進行排序…如此類推
    2. 默認按照升序（從小到大）排序
    3. asc升序
    4. desc降序

  • 示例1：將所有數據按年齡排序

  • select * from student order by age;

  • 示例2：將沒有地址的數據按年齡降序排序

  • select * from student where address="" or address=null order by age desc;

分頁

  • 格式：select * from 表名 limit start,count;
  • 說明：start索引從0開始
  • 示例：從第0個開始看，看三行
  • select * from student limit 0,3;

關聯

  • 建表語句

    1. create table class(id int auto_increment primary key,name varchar(20) not null, stuNum int not null);
    2. create table students(id int auto_increment primary key,name varchar(20) not null, gender bit default 1, classid int not null, foreign key(classid)references class(id));

  • 插入一下數據

    • insert into class values(0, "5A", 30),(0, "5B", 32),(0, "5C", 29),(0, "5D", 35);
    • insert into students values(0, "Jerry", 1, 4),(0, "Irene", 0, 3),(0, "Prinz", 0, 1);

  • 關聯查詢

    • select students.name,class.name from class inner join students on class.id=students.classid;
分類
  1. 表A inner join 表B
    • 表A與表B匹配的行會出現在結果集中
  2. 表A left join 表B
    • 表A與表B匹配的行會出現在結果集中，外加表A中獨有的數據，未對應的數據使用null填充
  3. 表A right join 表B
    • 表A與表B匹配的行會出現在結果集中，外加表B中獨有的數據，未對應的數據使用null填充
