發表於2020-09-01|更新於2020-09-03|計算機語言

NoSQL簡介

  • 全名：Not Only SQL，指的是非關係型的數據庫
  • 隨著訪問量的上升，網站的數據庫的性能出現了問題，於是nosql被設計出來

優點

  • 高可拓展性
  • 分佈式計算
  • 低成本
  • 架構的靈活性，半結構化的數據
  • 沒有複雜的關係

缺點

  • 沒有標準化
  • 有限的查詢功能（目前為止）
  • 最終一致是不直觀的程序

分類

列存儲

  • 代表
    • Hbase
    • Cassandra
    • Hypertable
  • 特點
    • 是按列存儲數據的。最大的特點是方便存儲結構化或半結構化數據，方便做數據壓縮，對針對某一列的或者幾列的查詢有非常大的IO優勢

文檔存儲

  • 代表
    • MongoDB
    • CouchDB
  • 特點
    • 文件儲存一般用類似json的格式存儲，儲存的內容是文檔型的。這樣也就有機會對某些字段建立索引，實現關係數據庫的某些功能

key-value存儲

  • 代表
    • Redis
    • Tokyo Cabinet/Tyrant
    • Berkeley DB
    • MemcacheDB
  • 特點
    • 可以通過key快速查詢到value。一般來說，存儲不管value的格式。（Redis包含了其他功能）

圖存儲

  • 代表
    • Neo4J
    • FlockDB
  • 特點
    • 圖形關係的最佳存儲，使用傳統關係數據庫來解決的話性能低下，而且設計使用不方便

對象存儲

  • 代表
    • db4o
    • Versant
  • 特點
    • 通過類似面向對象語言的語法操作數據庫，通過對象的方式存儲數據

xml數據庫

  • 代表
    • Berkeley DB XML
    • baseX
  • 特點
    • 高效的存儲XML數據，並支持XML的內部查詢語法，比如XQuery，Xpath

MongoDB

  • MongoDB是一個基於分佈式文件存儲的數據庫。由C++語言編寫。旨在爲WEB應用提供可擴展的高性能數據存儲解決方案。
  • MongoDB是一個介於關係數據庫和非關係數據庫之間的產品，是非關係數據庫當中功能最豐富，最像關係數據庫的。它支持的數據結構非常鬆散，是類似json的bson格式，因此可以存儲比較複雜的數據類型。Mongo最大的特點是它支持的查詢語言非常強大，其語法有點類似於面向對象的查詢語言，幾乎可以實現類似關係數據庫單表查詢的絕大部分功能，而且還支持對數據建立索引。

主要特點

  1. 面向集合存儲，易存儲對象類型的數據。
  2. 模式自由。
  3. 支持動態查詢。
  4. 支持完全索引，包含內部對象。
  5. 支持查詢。
  6. 支持複製和故障恢復。
  7. 使用高效的二進制數據存儲，包括大型對象（如影片等）。
  8. 自動處理碎片，以支持雲計算層次的擴展性。
  9. 支持 Golang，RUBY，PYTHON，JAVA，C++，PHP，C#等多種語言。
  10. 文件存儲格式爲BSON（一種JSON的擴展）。
  11. 可通過網絡訪問。

SQL與MongoDB概念

SQL術語/概念 MongoDB術語/概念 解釋/說明
database database 數據庫
table collection 數據庫表/集合
row document 數據記錄行/文檔
index field 數據字段/域
table joins 錶鏈接，MongoDB不支持
primary key primary key 主鍵，MongoDB自動將_id字段設置為主鍵

數據庫

  • 一個MongoDB中可以建立多個數據庫
  • MongoDB的單個實例可以容納多個獨立的數據庫，每一個有自己的集合和權限，不同的數據庫也放置在不同的文件中
  • 數據庫是通過名字來標識，數據庫名可以是滿足以下條件的任意UTF-8字符串
    • 不能是空字符串""
    • 不得含有空格、.(點)、$、\、/、\0(空字符)
    • 全部小寫
    • 最多64字節
  • 有一些數據庫名是保留的，可以直接訪問這些特殊作用的數據庫
    • admin：從權限的角度來看，這是root數據。要是將一個用戶添加到這個數據庫，這個用戶將自動繼承所以數據庫的權限
    • local：這個數據永遠不會被複製，可以用來儲存限於本地單台服務器的任意集合
    • config：當Mongo用於分片設置時，config數據庫在內部使用，用於保存分片的相關

文檔

概念

  • 文檔是MongoDB最核心的概念，是其核心單元
  • 多個鍵及其關聯的值有序的放置在一起就是文檔。MongoDB使用BSON來儲存數據
  • BSON可以理解為在JSON基礎上添加了一些json沒有的數據類型

注意

  • 文檔中的鍵值對是有序的
  • 文檔中的值不僅可以是雙引號內的字符串，還可以是其他幾種類型的數據（甚至可以是整個嵌入的文檔）
  • MongoDB區分類型和大小寫
  • MongoDB的文檔不能有重複的鍵
  • 文檔的鍵是字符串

文檔命名規範

  • 鍵不能含有\0(空字符)。這個字符用來表示鍵的結尾
  • 和$有特別的意義，只有在特定環境下才能使用
  • 以下劃線_開頭的鍵是保留的

集合

  • 集合是一組文檔的組合。
  • 在MongoDB中，集合是無模式的，也就是說集合中存儲的文檔結構可以是不同的。比如下面兩個文檔可以同時存在一個集合中
    • {"name":"Jerry"}
    • {"Name":"Jerry","gender":"male"}
  • 當一個文檔插入時，集合就會被創建
  • 合法的集合名字
    • 不能以system.開頭
    • 不能是空字符串""
    • 不能含有\0(空字符)。這個字符用來表示集合名的結尾

數據類型 描述

使用MongoDB

  1. 在目錄下創建data目錄，且在data目錄下創建db目錄和log目錄
  2. 打開命令行，進入MongoDB安裝地址的bin目錄下
  3. 輸入指令mongod.exe --dbpath=上面創建的db目錄路徑（不要關掉這個cmd窗口！）
  4. 再打開另一個命令行，進入MongoDB安裝地址的bin目錄下
  5. 連接MongoDB（出現<代表連接成功）
  6. 啟動後可以用可視化工具管理MongoDB：MongoDBChef

文檔操作

插入文檔

  1. 使用insert()方法插入文檔

    • 插入一個語法：db.集合名.insert(文檔)
    • 示例：db.student.insert({name:"Irene", age:17, gender: 0, city:"Paris", isDelete:0})
    • 插入多個語法：db.student.insert([{文檔1, 文檔2, ..., 文檔n}])
    • 示例：db.student.insert([{name:"Irene", age:17, gender: 0, city:"Paris", isDelete:0},{name:"Prinz Eugen", age:17, gender: 0, city:"Berlin", isDelete:0}])

  2. 使用save()方法插入文檔

    • 語法：db.集合名.save(文檔)
    • 說明：如果不指定_id字段，save()方法類似於insert()。如果指定_id字段，則會更新_id字段的數據。
    • 示例1：db.student.save({name:"Willams", age:17, gender: 1, city:"HK", isDelete:0})
    • 示例2（指定_id字段）：db.student.save({_id:ObjectId("5f4f725c69755bc92802e4e8"),name:"Willams", age:23, gender: 1, city:"HK", isDelete:0})

文檔更新

  1. update()方法用途更新已存在的文檔
    • 語法：
    • 參數說明：
      • query：update的查詢條件,類似於SQL裡update語句內的where後面的內容
      • update：update的對象和一些更新的操作符（$set,$inc）等，$set直接更新，$inc在原有的基礎上累加後更新
      • upset：可選，如果不存在update記錄，是否當新數據插入，true為插入，false為不插入，默認為false
      • multi：可選，MongoDB默認是false，只更新找到的第一條記錄，如果這個參數為true，就按照條件查找出來的數據全部更新
      • writeConcern：可選，拋出異常的級別
  • 例子：將Jerry的年齡更新為44
  • db.student.update({name:"Jerry"},{$set:{age:44}})
    • Jerry age值為44
  • 累加：db.student.update({name:"Jerry"},{$inc:{age:44}})
    • Jerry age值為88（累加）
  • 全改：db.student.update({name:"Jerry"},{$inc:{age:44}},{multi:true})
    • 表內所有Jerry 的age都改為了44

文件查詢

  1. find()方法
    • 查看集合下所有的文檔（數據）
      • 語法：db.集合名.find()
      • 示例：db.student.find()
  2. find()方法查詢指定列
    • 語法：
  • 參數說明：
    • query：查詢條件
    • key：要顯示的字段，1表示顯示
  • 示例（查看所有gender為1的文檔 name與age 的數據）：db.student.find({gender:1},{name:1,age:1})
  • 示例（查看所有的文檔 name與age 的數據）：db.student.find({},{name:1,age:1})
  1. pretty()方法以格式化的方式來顯示文檔
    • 示例：db.student.find().pretty()
  2. findOne()方法查詢匹配結果的第一條數據
    • 示例：db.student.findOne({gender:0})

查詢操作符

  • 作用：條件操作符用於比較兩個表達式並從MongoDB集合中獲取數據
    1. 大於操作符：$gt
      • 語法：db.集合名.find({<key>:{$gt:<value>}})
      • 示例（尋找age大於20的）：db.student.find({age:{$gt:20}})
    2. 大於等於操作符：$gte
      • 語法：db.集合名.find({<key>:{$gte:<value>}})
    3. 小於操作符：$lt
      • 語法：db.集合名.find({<key>:{$lt:<value>}})
    4. 小於等於操作符：$lte
      • 語法：db.集合名.find({<key>:{$lte:<value>}})
    5. 大於等於 和 小於等於 ：$gte 和 $lte
      • 語法：db.集合名.find({<key>:{$gte:<value>,$lte:<value>}})
    6. 等於：:
      • 語法：db.集合名.find({<key>:<value>})
    7. 使用_id進行查詢：
      • 語法：db.集合名.find({"_id":ObjectId("_id值")})
      • 示例：db.student.find({"_id" : ObjectId("5f4f6d5769755bc92802e4e7")})
    8. 查詢某個結果集的數據條數：
      • 語法：db.集合名.find().count()
      • 示例：db.student.find().count()
    9. 查詢某個字段的值中是否包含另一個值：
      • 語法：db.student.find({<key>:<value>})
      • 示例（查詢包含err的字段）：db.student.find({name:/err/})
    10. 查詢某個字段的值是否以另一個值開頭：
      • 示例（查詢以Je開頭的字段）：db.student.find({name:/^Je/})

條件查詢 and 和 or

  1. AND條件

    • 語法：db.集合名.find({條件1, 條件2, ..., 條件n})
    • 示例（所有gender為0且age大於16的字段）：
      • db.student.find({gender:0,age:{$gt:16}})

  2. OR條件

    • 語法：

      • db.student.find({$or:[{age:17},{age:{$gte:18}}]})

limit/skip

  1. limit()：讀取指定數量的數據記錄
    • 示例：db.student.find().limit(2)
  2. skip()：跳過指定數量的數據
    • 示例：db.student.find().skip(2)
  3. limit()與skip()聯合使用
    • 通常通過這種方式實現分頁功能
    • 示例：db.student.find().skip(3).limit(3)

排序

  • 語法：db.集合名.find().sort({<key>:1|-1})
  • 示例（升序）：db.student.find().sort({age:1})
  • 示例（降序）：db.student.find().sort({age:-1})
  • 注意：1表示升序，-1表示降序
